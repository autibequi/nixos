Quando Pedro diz "projeto real" ou "precisa dar certo", significa:

1. **Pesquisa extensiva** — usar todos os tokens disponiveis, disparar agentes em paralelo para pesquisa
2. **Dados reais** — nunca inventar numeros, sempre citar fonte (ABRAINC, FIPE ZAP, etc)
3. **Cenarios** — sempre otimista/realista/pessimista com probabilidades
4. **Graficos Mermaid** — visualizar tudo (fluxos, projecoes, quadrantes, jornadas)
5. **Links Obsidian** — interconectar documentos com `[[]]`
6. **Documento mastigado** — guia passo-a-passo com o que falar, onde clicar, templates prontos
7. **CUIDADOS** — documento separado com tudo que pode dar errado
8. **Stubs de pesquisa** — criar arquivos placeholder para pesquisas pendentes (agentes pegam depois)
9. **Organizacao final** — INDEX.md como ponto de entrada, pastas separadas (codigo/, docs/)
10. **Agentes autonomos** — config de agent para iterar automaticamente ate produto funcional

**Why:** Pedro prepara projetos para serem executados por terceiros (primo, parceiros). O material precisa ser auto-explicativo e completo o suficiente para alguem sem contexto pegar e executar.

**How to apply:** Usar como checklist sempre que criar material para projetos reais do Pedro.
